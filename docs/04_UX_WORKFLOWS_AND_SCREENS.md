## 7. Itinerary Builder

Three-pane concept:
- Left: days and structure
- Centre: editable itinerary canvas
- Right: catalogue, properties and pricing links

Key interactions:
- drag item
- duplicate item
- save as reusable component
- mark internal/customer-visible
- attach supplier or quote item
- version and restore
- preview PDF
- generate AI draft

## 12. Mobile Strategy

The initial product should be responsive but desktop-first.

Mobile priorities:
- view and assign leads
- add notes
- log calls
- update task or booking status
- view itinerary
- record payment reference
- receive alerts

Do not build a separate mobile application in the MVP.
