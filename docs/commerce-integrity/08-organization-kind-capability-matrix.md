# 08 — Organization kind capability matrix

Every kind must run **Acquire → Sell → Avail → Price → Operate → Finance → Retain** without the network.

| Kind | Offline OS depth (Integrity era) | Notes |
|------|----------------------------------|-------|
| travel_agency | Full ERP | Owns requirements, quotes, ops |
| hotel / homestay / farmstay | Stay OS | Shared stay core; overlays for home/farm |
| restaurant | Packages, capacity, kitchen, bill | Travel-group oriented |
| car_rental | Mobility Car OS 1.0 (rates, rental lifecycle, deposit/bill) | Must not be network-only |
| driver | Mobility Driver OS 1.0 (calendar, DriverJob, invoice/pay) | Thin OK; mobile-first |
| dmc | **Agency OS variant** (1.0) | B2B clients, net rates, sub-suppliers, settlements |
| platform | Catalog / moderation | Not a seller OS |

## DMC (Agency OS variant)

Agency OS core + DMC chrome: B2B Party workflows, quote net/sell, multi-service SR items to local suppliers, PartnerSettlement board. No parallel trip model.

## Homestay (beyond labels)

Entire-home vs room, house rules ack, host-present, flexible check-in — attributes on asset/reservation; same stay inventory DB.
